# PLAN: Add --version and --changelog to DCO-DailyCheck.py

## Description
Add two new CLI arguments:
- `--version`: prints current script version and exits.
- `--changelog`: prints a 3-column table (Version / Module / Change) and exits.

Data stored as a Python constant `CHANGELOG` inside the script.

---

## Module Category Convention

| Category | When to use |
|:---|:---|
| `PPCR`, `PPDM`, `DD`, `VC`, `ECS`, `OS10`, `IDRAC` | Change in a specific product module |
| `GETINFO` | Changes to the data collection phase (`*getinfo.py`) |
| `PROCESS` | Changes to the data processing phase (`*processinfo.py`) |
| `REPORTS` | Changes to report layout, ordering or structure |
| `SCRIPT` | Changes to the main script or CLI arguments |
| `CONFIG` | Changes to `DCOconfig.py` or config file structure |
| `COMMON` | Changes to shared utilities (`functions.py`, `DCOreport.py`) |
| `SECURITY` | SSL certificates, passwords, authentication |
| `EMAIL` | Email sending functionality |
| `ARCHIVING` | File rotation, archiving, cleanup |

## Proposed Changelog

```
Version    Module     Change

1.0        SCRIPT     Initial baseline version
1.01       PPCR       Improved job duration display to human-readable format (e.g. 1d 02h 30m)
           PPCR       Fixed a report generation error in the Protection Jobs table

1.04       REPORTS    Fixed display order of systems (Protection → Storage → Compute → Network)
1.05       VC         Updated datastore capacity thresholds (Green<85%, Yellow 85-95%, Red≥95%)
1.06       SCRIPT     Added --version and --changelog arguments
           REPORTS    Renamed and reorganized historical artifacts to match version numbers
```

## Proposed Code Changes — `src/DCO-DailyCheck.py`

**1. Constants (after imports):**
```python
__version__ = "1.06"

CHANGELOG = [
    ("1.0",  "SCRIPT",  "Initial baseline version"),
    ("1.01", "PPCR",    "Improved job duration display to human-readable format (e.g. 1d 02h 30m)"),
    ("",     "PPCR",    "Fixed a report generation error in the Protection Jobs table"),
    ("1.02", "PPCR",    "Updated Policies table color coding based on last backup date"),
    ("1.03", "GETINFO", "Improved compatibility when passing parameters to collection modules"),
    ("1.04", "REPORTS", "Fixed display order of systems (Protection → Storage → Compute → Network)"),
    ("1.05", "VC",      "Updated datastore capacity thresholds (Green<85%, Yellow 85-95%, Red≥95%)"),
    ("1.06", "SCRIPT",  "Added --version and --changelog arguments"),
    ("",     "REPORTS", "Renamed and reorganized historical artifacts to match version numbers"),
]
```

**2. Add arguments in `create_parser()`:**
```python
parser.add_argument('--version', action='store_true',
    help='Show the current script version and exit.')
parser.add_argument('--changelog', action='store_true',
    help='Show a table of all changes per version and exit.')
```

> [!NOTE]
> `action='store_true'` is used instead of argparse's built-in `action='version'` to control the output format and keep `-v` free for future use.

**3. Handle in `__main__` (after `args = parser.parse_args()`):**
```python
if args.version:
    print(f"DCO-DailyCheck v{__version__}")
    sys.exit()
if args.changelog:
    print(f"{'Version':<10} {'Module':<10} {'Change'}")
    print("-" * 90)
    for version, module, change in CHANGELOG:
        print(f"{version:<10} {module:<10} {change}")
    sys.exit()
```
