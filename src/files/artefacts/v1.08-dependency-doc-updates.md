# Artefacto v1.08 - Documentación de dependencias y entorno verificado

## Contexto
Para garantizar despliegues consistentes del paquete WinPython utilizado por los scripts de DCO, era necesario documentar la dependencia adicional `pyvmomi` y proporcionar un listado reproducible de versiones probadas.

## Cambios propuestos/realizados
1. **Archivo `requirements-verified.txt`** en la raíz del repositorio con las versiones verificadas (Python 3.12.10 + cryptography 45.0.7, openpyxl 3.1.5, pandas 2.3.2, requests 2.32.5, urllib3 2.5.0, jinja2 3.1.6, pyvmomi 9.0.0.0). Incluye instrucciones para instalarlo con `pip install -r requirements-verified.txt`.
2. **README.md**: se añade `pip install pyvmomi` al listado de módulos necesarios y una nota explicando cómo usar el nuevo requirements para replicar el entorno validado.
3. **`DCO-DailyCheck.py`**: incremento de versión a `1.08` y entrada en el changelog bajo la categoría DOCS para reflejar la documentación y el listado verificado.

## Pruebas sugeridas
- En un entorno WinPython limpio, ejecutar `pip install -r requirements-verified.txt` y verificar que los módulos se importan correctamente.
- Revisar el README renderizado para confirmar que el bloque de instalación y la nota se muestran con formato correcto.
