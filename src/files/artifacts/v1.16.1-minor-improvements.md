# Feature Release — v1.16.1 (Minor Improvements)

**Date:** 2026-06-12  
**Branch:** feature/v1.16.1-minor-improvements  
**Version:** v1.16.1  

## Implemented Changes

### 1. Data Domain Capacity Precision
- Increased the precision of the capacity used percentage (Active Tier, Cloud Tier, and Combined) from 1 decimal to 2 decimals in Data Domain reports (`reportDC` and `reportDCI`).
- **Files Modified:**
  - `src/DD/DDprocessinfo.py`
  - `src/DD/DDcreatereportDCI.py`

### 2. IDRAC Warning Color Fix
- Fixed the "Log Status" color for "Warning" in IDRAC `reportDC` to use the standard pastel yellow (`PASTEL_YELLOW`) instead of pastel orange.
- **Files Modified:**
  - `src/IDRAC/IDRACcreatereportDC.py`

### 3. Data Domain Alert Color Fix
- Created a specific color mapping function (`colorByAlertStatus`) for Data Domain "System Alerts" in `reportDC`.
- Ensures that lower priority alerts ("WARNING", "NOTICE", "INFO", "DEBUG") are correctly displayed in pastel yellow (`PASTEL_YELLOW`), while "OK" remains green and critical alerts default to red.
- Renamed the generic `colorByStatus` to `colorByFilesystemStatus` for clarity.
- **Files Modified:**
  - `src/DD/DDcreatereportDC.py`
  - `src/DCO-DailyCheck.py` (Changelog updated)

### 4. Data Domain ReportDCI Crash Fix
- Fixed a `ValueError` crash in `DDcreatereportDCI.py` caused by `color_alertsDetail` returning `None` for unhandled severity levels (such as "OK" or "Clear").
- Added a default return value `[''] * len(row)` to ensure pandas `apply(axis=1)` always receives a list-like object.
- **Files Modified:**
  - `src/DD/DDcreatereportDCI.py`
  - `src/DCO-DailyCheck.py` (Changelog updated)
