# MISSION: Increase Data Domain Replica Collection Limit

**Date:** 2026-04-16 16:35 UTC  
**Source Branch:** feature/dd-replica-limit-fix  
**Change Type:** improvement / fix  
**Previous Version:** v1.11.0

## Objective / Mission
The goal was to resolve a data truncation issue in the Data Domain (`DD`) module. The previous implementation used the default REST API behavior, which limited the number of returned replication contexts to 20. This caused incomplete reports in environments with more than 20 MTree replications.

## Implementation Details
1.  **Modified File**: `src/DD/DDgetinfo.py`
2.  **Function Updated**: `dd_get_replicas`
3.  **Changes**:
    *   Defined a `params` dictionary with `size: '100'` and `page: '0'`.
    *   Updated the `requests.get` call to pass these parameters to the Data Domain API.
    *   This ensures that up to 100 replication contexts are collected in a single scan.

## Impact and Benefits
- **Complete Visibility**: Reports now show all replications (up to 100) instead of being truncated at 20.
- **Improved Accuracy**: Dashboards and investigation reports now reflect the actual state of all replication MTrees.
