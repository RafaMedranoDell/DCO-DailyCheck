# MISSION: iDRAC Report Simplification

**Date:** 2026-04-17  
**Branch:** feature/idrac-report-simplification  
**Version:** v1.14.0  
**Previous Version:** v1.13.0

## Objective
Streamline the iDRAC monitoring reports to provide high-level visibility at a glance and focus deeper investigations only on identified issues.

## Implementation Details

### 1. DC Report (Summary)
The iDRAC summary table has been reduced from 8 rows to **3 consolidated rows**, aligning with the iDRAC Panel visual hierarchy:
- **System Health**: Worst status aggregated from Chassis, System, Processors, Power Supplies, Fans, and Temperatures.
- **Storage Health**: Health status of the storage controllers.
- **Logs Status**: `OK` if no new log entries were detected in the period; otherwise reflects the worst `Severity` found.

### 2. DCI Report (Investigation)
Detailed tables are now **filtered by default** to eliminate noise:
- **Hardware Tables**: Chassis, System, Processors, PSU, Fans, Storage, and Thermal tables now only display rows where `Health != OK` or `Status != Enabled`. If all components are healthy, the table is omitted from the report.
- **LifeCycle Controller Log**: Now specifically filtered to only show entries where `Severity != OK`.
- **Visual Alerts**: Implemented row-level color coding for all logs (SEL, LC, Faults) using the standardized `DCOreport` styling: **Critical = Red**, **Warning = Yellow**.

### 3. Processing Logic
- `IDRACprocessinfo.py` updated to compute the aggregate `System Health` and `Logs Status`.
- `IDRACcreatereportDC.py` updated for the 3-row layout.
- `IDRACcreatereportDCI.py` updated with row-level filtering and dynamic table naming (e.g., "Chassis (Problems)").

## Benefits
- **Faster Triage**: Dashboard reveals the overall health of 6 subsystems in a single row.
- **Reduced Overhead**: Investigation reports are significantly shorter and focused on actionable errors.
- **Improved UI Consistency**: Naming and hierarchy now closer to the native iDRAC Dashboard.
