# Feature Release — v1.16.0 (Auto-update SSL Certificates)

**Date:** 2026-06-04  
**Branch:** feature/v1.16.0-auto-update-ssl-certs  
**Version:** v1.16.0  

## Implementation Plan: Auto-update SSL Certificates in getinfo

This plan describes the architecture to allow `getinfo` modules to verify and automatically update SSL certificates if they change, avoiding the need to run `config_tool.py --certs update` manually.

## Security Analysis

> [!NOTE]
> **Security Analysis (Low Risk)**
> The original purpose of verifying the certificate signature (hash) is to ensure we are connecting to the legitimate server and to prevent Man-in-the-Middle (MitM) attacks. By auto-updating the certificate, we lose this strict protection.
> 
> However, since **the credentials used by the script are Read-Only**, the impact of an attacker intercepting the connection is minimal (they would only gain read access to the systems, without the ability to destroy backups or alter configurations). 

**Global Toggle (Mitigation):**
To balance operational convenience and strict security, we will implement a global toggle `"auto_update_certs": false` in the configuration. The auto-update will only occur if this flag is explicitly set to `true`. Otherwise, the script will maintain strict security and fail.

## Proposed Changes

The implementation will focus on code reuse (`common` folder) to minimize the impact on individual modules.

### Configuration Layer

#### [MODIFY] `src/TEMPLATEconfig.json`
Add the `"auto_update_certs": false` toggle to the root of the template configuration.

### Common Layer

#### [MODIFY] `src/common/functions.py`
Move the `scalar_first` and `save_cfg` functions from `config_tool.py` here to centralize JSON formatting logic.
Modify the validation function to perform the auto-update if the initial check fails:
- Update `valid_certificate_fingerprint(host, port, cert_hash, dcocfg=None, system=None, instance=None)`.
- If the hash does not match, and `dcocfg`, `system`, and `instance` are provided:
  - Check `dcocfg.config.get("auto_update_certs", False)`.
  - If `True`, log a warning: `logger.warning("{system}/{instance} certificate hash changed. Updating automatically.")`, call `dcocfg.update_certificate_hash()`, and return `True`.
  - If `False`, log a critical error and return `False`.

#### [MODIFY] `src/common/DCOconfig.py`
Add the capability to save the configuration directly from the `DCOconfig` object:
- Modify `__init__` to store the configuration file path.
- Create a new method `update_certificate_hash(self, system, instance, new_hash)` that:
  1. Locates the instance in the dictionary loaded in memory.
  2. Updates the `certHash` value.
  3. Saves the modified JSON file to disk using `fn.save_cfg`.

### Getinfo Modules

#### [MODIFY] All `*getinfo.py` scripts
To leverage the new centralized logic, each module that calls `fn.valid_certificate_fingerprint` must pass the necessary objects so it can auto-update.
Modules to modify: `PPCRgetinfo.py`, `DDgetinfo.py`, `ECSgetinfo.py`, `PPDMgetinfo.py`, `VCgetinfo.py`, `OS10getinfo.py`, `ESXgetinfo.py`, and `TEMPLATEgetinfo.py`.

Example of required change in each:
```diff
- if not fn.valid_certificate_fingerprint(instance, api_port, cert_hash):
+ if not fn.valid_certificate_fingerprint(instance, api_port, cert_hash, dcocfg, system, instance):
```


### Automated Tests
1. Run the collection script (`DCO-DailyCheck.py`) with `"auto_update_certs": false`.
2. Simulate a certificate change by manually altering the `certHash` of an instance in `config_encrypted.json` to an incorrect value.
3. Verify the script fails securely.
4. Set `"auto_update_certs": true` and run again. Verify that:
   - The script **does not fail**.
   - The auto-update message is shown in the log (`WARNING`).
   - The `config_encrypted.json` file is updated with the correct hash on disk, preserving its formatting.

### Interactive Prompt Enhancements

When using `config_tool.py --interactive`, the boolean key `auto_update_certs` is now prompted with the current value displayed, e.g.:

```
Enter auto_update_certs [True] (True/False):
```

Only the inputs `True`, `False` (case‑insensitive) or pressing **Enter** (to keep the current value) are accepted. Any other input triggers an error message and the prompt repeats until a valid entry is provided.

This ensures the configuration cannot be set to an invalid value interactively.
