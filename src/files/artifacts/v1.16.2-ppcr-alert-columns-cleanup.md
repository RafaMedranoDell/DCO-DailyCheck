# MISSION: Remove Non-Existent Columns from PPCR Alert Detail Table

**Date:** 2026-06-22  
**Source Branch:** fix/ppcr-dci-tables-improvements  
**Change Type:** fix / cleanup  
**Previous Version:** v1.16.1

## Objective / Mission
Remove non-existent columns "Type" and "Modified Date" from the Alert Detail table in the PPCR module. These columns were referenced in the code but did not exist in the original JSON data source, causing potential confusion and code inconsistency.

## Analysis / Background
The PPCR module's `process_alerts()` function in `PPCRprocessinfo.py` was attempting to process and display columns "Type" and "Modified Date" in the Alert Detail table. However, these fields were not present in the original JSON data retrieved from the PPCR API (only "category", "createdByService", "creationDate", "acknowledged", "severity", "summary", "description", "remedy" were available). This created a code inconsistency where the CSV generation tried to access non-existent fields.

## Implementation Details
The fix involved two specific changes in `src/PPCR/PPCRprocessinfo.py`:

1. **Removed non-existent columns from selected_columns dictionary** (lines 30-37):
   - Removed `'type': 'Type'` 
   - Removed `'modifiedDate': 'Modified Date'`
   - Kept only valid columns: Category, Severity, Creation Date, Acknowledged, Summary, Remedy

2. **Updated timestamp conversion** (line 41):
   - Changed `fn.df_timestamps_to_dates(active_alerts, ["creationDate", "modifiedDate"])`
   - To `fn.df_timestamps_to_dates(active_alerts, ["creationDate"])`

This approach ensures the CSV generation only processes fields that actually exist in the JSON data, eliminating the reference to non-existent columns.

## Modules / Files Updated
- **src/PPCR/PPCRprocessinfo.py**
  - Removed `'type': 'Type'` and `'modifiedDate': 'Modified Date'` from selected_columns dictionary
  - Updated timestamp conversion to only process "creationDate"
  - This affects the process_alerts() function that generates the alertDetail CSV

## Impact and Benefits
- **Code Consistency**: CSV generation now only references fields that exist in the source JSON
- **Cleaner Output**: Alert Detail table displays only valid columns without empty/non-existent fields
- **No Side Effects**: Change is purely cleanup, no functional impact on existing behavior
- **Maintainability**: Reduces confusion about data sources and field availability
