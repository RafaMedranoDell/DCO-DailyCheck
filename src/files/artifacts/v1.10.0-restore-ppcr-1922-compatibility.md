## Objective / Mission
The primary objective was to resolve authentication failures occurring after the upgrade to **PowerProtect Cyber Recovery (PPCR) 19.22**. The mission aimed to ensure the automation scripts correctly handle the modernized IAM service, specifically addressing endpoint shifts and case-sensitivity requirements for usernames.

## Analysis / Background
Research and field logs revealed that the 19.22 version introduces a stricter Identity Management (IAM) service:
1.  **Endpoint Authentication**: Connection must be routed through the `/v9/auth/login` endpoint.
2.  **Username Normalization**: Unlike previous versions, the 19.22 API enforces strictly **lowercase** characters for standard local service accounts (e.g., `dcooper`). Older configurations using mixed-case (`DCOoper`) were triggering "Unauthorized" errors (IAM3007).
3.  **Upgrade Side Effects**: System upgrades often invalidate stored SSL certificate fingerprints, leading to "hash mismatch" errors during data collection.

## Implementation Details
The solution implements a robust, multi-layered approach to handle these changes:
- **Case-Insensitive Automation**: Modified the `PPCRapi` module to automatically convert usernames to lowercase before sending the login request. This eliminates the need for manual configuration updates for existing users while ensuring compliance with the IAM service.
- **Diagnostic Transparency**: Enhanced 401 error handling to log the raw server response, assisting administrators in identifying if an authentication failure is due to credentials or an account lockout.
- **Centralized Documentation**: Added a dedicated "Known Issues & Troubleshooting" section to the main **README.md**. This section provides clear guidance on resolving common upgrade issues, such as certificate updates using the `config_tool.py`.
