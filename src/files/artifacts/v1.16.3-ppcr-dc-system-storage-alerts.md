# MISSION: Add Storage Alerts to PPCR DC System Alerts Table

**Date:** 2026-06-23  
**Source Branch:** feature/ppcr-dc-system-storage-alerts  
**Change Type:** feature / enhancement  
**Previous Version:** v1.16.2

## Objective / Mission
Add storage alerts to the PPCR DC System Alerts table, changing it to "System & Storage Alerts" to provide a more comprehensive view of alerts affecting both system and storage categories. This enhancement adds granular calculations for storage alerts while maintaining existing system and security alert calculations.

## Analysis / Background
The PPCR DC report table previously only displayed "System Alerts" which included alerts from the "system" category. However, the PPCR system also generates alerts in the "storage" category that were not being represented in the summary table. This created an incomplete view of the alert status, as storage-related alerts could be critical but were not visible in the main DC report.

The enhancement adds:
- Individual calculation of storage_status for future use
- Combined calculation of system_storage_status for current reporting
- Maintains existing system_status and security_status calculations
- Provides flexibility for future enhancements while solving the current visibility gap

## Implementation Details
The enhancement involved specific changes in two files:

**PPCRprocessinfo.py - process_alerts() function (lines 38-41, 49):**
1. **Added storage_status calculation:**
   - New calculation for alerts with category "storage"
   - Uses same severity logic as system alerts (Critical, Warning, OK)

2. **Added system_storage_status calculation:**
   - Combines alerts from both "system" and "storage" categories
   - Uses .isin() method to filter multiple categories
   - Provides the combined status for the DC report table

3. **Updated return statement:**
   - Changed from 2 values to 4 values: system_storage_status, security_status, system_status, storage_status
   - Provides all individual calculations for future flexibility

**PPCRprocessinfo.py - proccess_info() function (line 287):**
1. **Updated alert status reception:**
   - Changed from 2 values to 4 values
   - Updated na_count parameter from 2 to 4

**PPCRprocessinfo.py - instance_summary (line 303):**
1. **Updated table label:**
   - Changed from "System Alerts" to "System & Storage Alerts"
   - Uses system_storage_alerts_status (combined value)

**PPCRcreatereportDC.py - create_DC() function (line 69):**
1. **Updated coloring rule:**
   - Changed from ("System Alerts", colorAlertsBySeverityVal)
   - To ("System & Storage Alerts", colorAlertsBySeverityVal)

## Modules / Files Updated
- **src/PPCR/PPCRprocessinfo.py**

  - Added system_storage_status combined calculation in process_alerts()
