# MISSION: Deployment Process Improvements (v1.14.1)

**Date:** 2026-04-21  
**Branch:** feature/v1.14.1-deploy-fix  
**Version:** v1.14.1  
**Previous Version:** v1.14.0

## Objective
Optimize the automated deployment process to ensure all monitoring modules are included and generated packages are correctly versioned for easier tracking.

## Changes

### 1. Inclusion of ESX Module
- **Issue**: The recently created `ESX` standalone module was missing from the automated ZIP package generation.
- **Fix**: Added `'ESX'` to the `product_list` in `tools/deploy-pack.py`. 
- **Result**: All 5 standard module files (`__init__.py`, `getinfo`, `processinfo`, `reportDC`, `reportDCI`) are now packaged correctly.

### 2. Version-Prefixed Packages
- **New Feature**: Deployment ZIP files now include the current code version as a prefix.
- **Implementation**: 
    - `tools/deploy-pack.py` now dynamically reads the `__version__` variable from `src/DCO-DailyCheck.py` using regex.
    - Filename pattern: `{version}_deploypack-{timestamp}.zip`.
    - Example: `v1.14.1_deploypack-20260421_1650.zip`.

## Verification Details
- Successfully extracted version `v1.14.1` from source code during script execution.
- Verified that all modules (DD, ECS, ESX, iDRAC, OS10, PPCR, PPDM, VC) are processed.
- Python syntax check passed for all modified scripts.

## Benefits
- **Reliability**: Ensures clients receiving the deployment package have the latest ESXi standalone monitoring capabilities.
- **Traceability**: Package filenames now clearly state the version they contain, eliminating confusion during manual deployments or upgrades at client sites.
