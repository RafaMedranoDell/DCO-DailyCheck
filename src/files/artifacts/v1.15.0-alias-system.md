# Alias System Release — v1.15.0

**Date:** 2026-04-21  
**Branch:** feature/v1.15.0-alias-system  
**Version:** v1.15.0  

## Overview
This version introduces the **Alias System**, allowing users to define friendly names for monitored systems. Instead of seeing technical IP addresses in report headers, users can now see names like "Production vCenter" or "Backup DD".

## Key Changes

### 1. Configuration (TEMPLATEconfig.json)
- Added an optional `alias` field to all instance definitions.
- If left empty, the system automatically falls back to the `hostname`.

### 2. Management Tool (config_tool.py)
- **`--add`**: Now prompts for an optional alias when adding new instances.
- **`--modify`**: Interactive modification of aliases for existing instances.
- **`--list`**: Displays aliases in parenthesis next to hostnames for quick identification.

### 3. Integrated Reports
- **Consolidated Tables (DC)**: Modules using `systemSummary` (DD, ECS, iDRAC, OS10, PPCR, PPDM) now automatically display aliases in their column headers for a cleaner executive summary.
- **Detailed Investigation (DCI)**: Updated all 8 major modules to use the format `hostname (alias)`. This ensures technical precision while keeping the friendly reference.
- **Unified Headers**: Standardized report indexing by removing redundant "Instance" prefixes.

## How to use
Run the configuration tool to add aliases to your existing systems:
```bash
python config_tool.py --modify IDRAC/192.168.1.10
```
Then simply run your daily check as usual. The generated HTML reports will reflect the new names.

## Technical Details
- Added `get_instance_alias`, `get_instance_display_name`, and `get_instance_full_name` to `DCOconfig.py`.
- Enhanced `fn.systemSummary` in `functions.py` with column renaming logic.
- Maintains 100% backward compatibility with configurations missing the `alias` field.
