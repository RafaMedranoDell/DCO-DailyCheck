# MISSION: ESX Standalone Module

**Date:** 2026-04-17  
**Branch:** feature/esx-standalone-module  
**Version:** v1.13.0  
**Previous Version:** v1.12.0

## Objective
Create a new `ESX` module to monitor standalone ESXi hosts (without vCenter) using the vSphere SOAP API via `pyVmomi`.

## Implementation

### `src/ESX/__init__.py`
Empty package marker.

### `src/ESX/ESXgetinfo.py`
- Connects directly to ESXi SOAP API (`SmartConnect`).
- Collects: host health (`overallStatus`), datastores, VMs, triggered alarms.
- Reuses `fetch_soap_alarms()` imported from `VC.VCgetinfo` — no code duplication.

### `src/ESX/ESXprocessinfo.py`
- Mirrors `VCprocessinfo.py` structure.
- Maps health status (green/yellow/red/gray → OK/Warning/Critical/Unknown).
- Generates: `systemSummary.csv`, `datastoreStatus.csv`, `vmStatus.csv`, `alertsDetail.csv`.
- Same alarm and datastore thresholds as VC module.

### `src/ESX/ESXcreatereportDC.py`
- Adds summary table under section `"Compute" / "ESX"`.
- Rows: Host Health, Datastore Capacity, Active Alerts (same coloring as VC).

### `src/ESX/ESXcreatereportDCI.py`
- Table 1: **Datastore Capacity Detail** — all datastores, colour-coded by % Used.
- Table 2: **VM Power Status Detail** — VMs not `poweredOn`, highlighted red.
- Table 3: **Alerts Detail** — same columns as VC module.

### `src/TEMPLATEconfig.json`
Added `ESX` system block with JSON/CSV file mappings.

### `src/DCO-DailyCheck.py`
